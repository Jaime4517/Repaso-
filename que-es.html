¿Qué es un sitio web?

💡 Una página web es un documento que se muestra en Internet y puede contener texto, imágenes, videos o enlaces.

🖥️ Un sitio web es un conjunto de páginas web relacionadas entre sí, que comparten un mismo dominio.

🌍 Internet funciona conectando millones de dispositivos mediante redes. Cuando visitamos un sitio web, nuestro navegador pide información a un servidor que la envía a través de Internet.

🔎 Los navegadores interpretan el código de la web y lo muestran en pantalla. Los servidores almacenan las páginas y responden a estas peticiones.

📡 La información viaja por la red en paquetes de datos que se transmiten mediante cables, fibra óptica o redes inalámbricas.

Conceptos importantes:

  • 🌐 1. Dirección IP

    Una dirección IP es como la dirección postal de un dispositivo en Internet. Cada móvil, ordenador o servidor conectado a la red tiene una IP para poder enviar y recibir información. Ejemplo: 192.168.1.15 o en formato moderno IPv6: 2001:0db8:85a3:0000:0000:8a2e:0370:7334

    📌 ¿Para qué sirve?

    Sirve para identificar: Tu ordenador Una web Un servidor Un router Cuando visitas una página web, tu dispositivo necesita saber: “¿A qué dirección envío mi petición?” Y usa la IP.

    🧠 Ejemplo real

    Cuando entras en una web: www.google.com En realidad tu ordenador termina comunicándose con una IP parecida a: 142.250.184.206

  • 🧭 2. DNS

    El DNS (Domain Name System) es como la agenda telefónica de Internet. Convierte nombres fáciles de recordar: google.com en direcciones IP reales: 142.250.184.206

    📌 ¿Por qué existe?

    Porque los humanos recordamos nombres fácilmente, pero las máquinas trabajan con IPs.

    🔄 Cómo funciona

    Escribes: youtube.com. Tu ordenador pregunta al DNS: “¿Qué IP tiene youtube.com?” El DNS responde: 142.251.xxx.xxx Tu navegador ya puede conectarse al servidor correcto.

  • 🔒 3. Protocolos HTTP y HTTPS

    Son las reglas que permiten que navegador y servidor se entiendan.

    📡 HTTP

    Significa: HyperText Transfer Protocol. Es el protocolo básico para cargar páginas web.

    Cuando visitas una web, tu navegador hace peticiones HTTP como:

    GET /index.html

    ❌ Problema de HTTP

    Los datos viajan “en abierto”.

    Eso significa que alguien podría interceptarlos.

    🔐 HTTPS

    La versión segura de HTTP.

    La “S” significa: Secure

    Usa cifrado mediante SSL/TLS.

    📌 ¿Qué protege?

    • Contraseñas
    • Tarjetas bancarias
    • Chats
    • Datos personales

    🧠 Ejemplo

    HTTP:

    http://ejemplo.com 🔓 Sin cifrar.

    HTTPS:

    https://ejemplo.com 🔒 Cifrado y seguro.

  • 📡 4. Servidores y clientes

    Internet funciona principalmente con este modelo: Cliente ⇄ Servidor

    💻 Cliente

    Es quien solicita información.

    Ejemplos:

    • Tu navegador
    • Tu móvil
    • Una app

    Cuando haces clic en una web, el cliente pide datos.

    🖥️ Servidor

    Es el ordenador que responde.

    Guarda:

    • páginas web
    • imágenes
    • bases de datos
    • vídeos

    y las envía cuando alguien las solicita.

    🧠 Ejemplo completo

    Tú escribes: netflix.com

    Entonces ocurre esto:

    • Tu navegador (cliente) pregunta al DNS.
    • Obtiene la IP de Netflix.
    • Se conecta al servidor.
    • Envía una petición HTTPS.
    • El servidor responde enviando:
      • HTML
      • imágenes
      • vídeos
      • datos
    • Tu navegador muestra la página.

  • 📦 5. Transferencia de datos por la red

    Cuando navegas por Internet, los datos NO viajan como un bloque gigante.

    Se dividen en pequeños fragmentos llamados:

    • Paquetes

      📦 Cada paquete contiene:

      • Parte de la información
      • Dirección origen
      • Dirección destino
      • Orden del paquete
    • 🚚 Viaje de los paquetes

      Los paquetes pasan por:

      • routers
      • cables
      • antenas
      • proveedores de Internet
      • servidores

      hasta llegar al destino.

      🧠 Ejemplo simple

      Si ves un vídeo de YouTube:

      • El vídeo se divide en miles de paquetes.
      • Los paquetes viajan por Internet.
      • Tu dispositivo los recibe.
      • Los vuelve a unir.
      • Ves el vídeo continuamente.

      🔗 Cómo se conecta todo

      Aquí tienes el flujo completo simplificado:

      1. Escribes una web

      ↓

      2. DNS busca la IP

      ↓

      3. Cliente se conecta al servidor

      ↓

      4. Usa HTTP/HTTPS

      ↓

      5. Los datos viajan en paquetes

      ↓

      6. La web aparece en pantalla

Tipos de paginas web

  • Estaticas:

    Son aquellas que no cambian. Siempre muestran el mismo contenido.

    Se realizan en lenguaje HTML.

  • Dinamicas:

    Son paginas que daran un resultado diferente cada vez que entres sin alterar el codigo dependiendo de la actuacion del usuario.

    Para realizarlo, combina:

    • HTML
    • Lenguajes dinamicos: ASP.NET, Java, PHP o JavaScript.
